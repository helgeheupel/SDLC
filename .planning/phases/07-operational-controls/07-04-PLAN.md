**YAML frontmatter:** Same pattern (document-id HH-ISMS-PROC-007, title "Internal Audit Program", date 2026-02-25, classification Internal, parent-document HH-ISMS-POL-001).

**Section structure (decision-tree pattern):**

1. **Purpose and Scope** — References ISO 27001 cl. 9.2 and ISO 42001 cl. 9.2. Scope: all ISMS and AIMS controls as defined in the Statements of Applicability (HH-ISMS-SOA-001 with 93 controls, HH-AIMS-SOA-001 with 38 controls). Combined IMS audit approach per IMS Context (HH-IMS-CTX-001).

2. **Normative References** — Table: ISO 27001 cl. 9.2, ISO 42001 cl. 9.2, ISO 19011:2018 (audit guidelines), IMS Context, SoA (both), governance model. Related Procedures (HH-ISMS-PROC-008 for audit results as management review input, HH-ISMS-PROC-001 for audit findings that constitute incidents).

3. **Audit Program Structure** — Per locked decision: one full-scope internal audit before Stage 1 covering all ISMS + AIMS controls.
   - **Audit cycle:** Annual (first audit must be complete before Stage 1 certification audit).
   - **Audit scope:** Full scope — all 93 ISO 27001 Annex A applicable controls + all 38 ISO 42001 Annex A applicable controls. Both SoAs define the audit universe.
   - **Audit type:** Combined IMS audit (single audit covering both ISMS and AIMS simultaneously per IMS Context approach).
   - Decision tree: Annual audit cycle -> Define scope from SoAs -> Select auditor -> Plan audit -> Execute audit -> Report findings -> Track corrective actions -> Feed into management review.

4. **Auditor Competence and Independence** — Per locked decision:
   - **Lead Auditor:** External CISO (NorthGRC) — provides independence per ISO 27001 cl. 9.2 requirement ("Auditors shall not audit their own work").
   - **NorthGRC qualifications:** ISO 27001 Lead Auditor certification, GRC platform expertise, external to HH operational processes.
   - **CISO Agent role:** Assists with evidence gathering (collecting documents, generating control status reports from SoA) but does NOT audit its own outputs. The CISO Agent produced most IMS documentation — NorthGRC must independently verify these artifacts.
   - **Independence boundary rule:** Any evidence item produced by the CISO Agent must be independently verified by NorthGRC. The audit checklist marks evidence source so NorthGRC knows what requires independent assessment.

5. **Audit Planning** — For each audit cycle:
   - Review previous audit results and open corrective actions.
   - Review risk assessment to prioritize high-risk areas.
   - Determine audit scope (full or focused — first audit is always full).
   - Prepare audit schedule (typically 2-3 days for full-scope IMS audit at Startup scale).
   - Prepare audit notification to auditees (CEO, CTO, IT Pentester).
   - Prepare evidence request list organized by control group.

6. **Audit Execution** — Process:
   - **Opening meeting:** NorthGRC presents scope, methodology, schedule to CEO + CTO.
   - **Evidence collection:** Document review (policies, procedures, records), interview (CEO, CTO, IT Pentester), observation (system configurations, access controls), automated evidence (Git logs, deployment logs, monitoring data).
   - **Control assessment:** For each applicable control in the SoA, assess implementation status and operational effectiveness using the scoring criteria below.
   - **Closing meeting:** NorthGRC presents preliminary findings to CEO + CTO.

7. **Audit Finding Classification** — From research:
   - **Conforming:** Control implemented and operating effectively. Evidence current and complete. Response: none.
   - **Opportunity for Improvement (OFI):** Conforming but could be more effective. Response: recommendation, address within next review cycle.
   - **Minor Nonconformity:** Partially implemented or evidence incomplete. No immediate risk. Response: corrective action within 30 days, root cause analysis.
   - **Major Nonconformity:** Not implemented or significant gap undermining ISMS/AIMS effectiveness. Response: immediate corrective action, CEO notification, must resolve before Stage 1.

8. **Scoring Criteria** — Per control:
   - 3: Fully conforming — evidence complete, current, demonstrates operational effectiveness.
   - 2: Partially conforming — control exists but evidence gaps or operational weaknesses.
   - 1: Minimally conforming — significant gaps, meets intent but not fully operational.
   - 0: Non-conforming — not implemented or no evidence.

9. **Audit Checklist Template** — Provide a representative sample showing the checklist structure (not all 131 controls, but the template):
   - Table columns: Control ID, Control Name, Evidence Required, Evidence Source (with flag: "CISO Agent output — requires NorthGRC independent review" vs "System-generated — automated evidence" vs "Human-produced"), Auditor Score (0-3), Finding Classification, Notes.
   - Sample rows for 5-6 controls across different categories (e.g., A.5.1 IS policies, A.5.24 Incident management, A.8.32 Change management, ISO 42001 A.2.2 AI Policy, A.5.4 AI impact assessment, A.10.3 Third-party AI).
   - Full checklist to be generated from SoA prior to each audit (NorthGRC + CISO Agent collaboration).

10. **Corrective Action Tracking** — For each nonconformity:
    - NCR (Nonconformity Report) created in Git: `audits/NCR-{YYYY}-{NNN}.md`
    - Fields: control ID, finding description, classification (minor/major), root cause analysis, corrective action plan, responsible person, target date, verification method.
    - CTO responsible for technical corrective actions. CEO responsible for governance corrective actions.
    - NorthGRC verifies corrective action effectiveness at next audit or interim review.
    - Corrective action status feeds into management review (HH-ISMS-PROC-008) input d.1.

11. **Annex A Control Mapping** — This procedure itself satisfies ISO 27001 cl. 9.2 and ISO 42001 cl. 9.2 (these are clauses, not Annex A controls). Note: no Annex A control directly maps to "internal audit" — it is a mandatory management system requirement, not an Annex A control.

12. **Regulatory Obligations** — ISO 27001 cl. 9.2, ISO 42001 cl. 9.2, TISAX (ISA Chapter 8 audit requirements), NIS2 (informational — supervisory audits in Growth).

13. **Document Control** — Version history and approval.

**CRITICAL anti-patterns:**
- Do NOT create a checklist where the CISO Agent audits its own outputs without flagging
- The "Evidence Source" column in the checklist is ESSENTIAL for independence
- NorthGRC leads — CISO Agent assists. This distinction must be clear throughout.
