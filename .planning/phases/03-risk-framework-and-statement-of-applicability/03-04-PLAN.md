Create `soa/scripts/generate-soa.js` based on the research code example (Pattern 5). The script MUST:
1. Read all `soa/A.*.yml` files
2. Sort by control number (A.5.1, A.5.2, ..., A.8.34) using numeric sorting
3. Group by theme (organizational, people, physical, technological)
4. Calculate summary statistics (total, applicable, not-applicable, by implementation status)
5. Generate a Markdown document with YAML frontmatter (document-id: HH-ISMS-SOA-001)
6. Include per-theme tables with columns: Control | Name | Applicable | Status | Justification
7. Truncate justification to first 2 sentences in the table (full justification available in per-control files)
8. Add a Risk-to-Control Traceability section listing which risks map to which controls
9. Add CEO-SIGNOFF-REQUIRED placeholder at the end
10. Write output to `soa/SoA-Consolidated.md`

After creating the script, run `cd soa/scripts && npm install && node generate-soa.js` to generate the initial `SoA-Consolidated.md`.
